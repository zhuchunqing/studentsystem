# 学生教育管理系统

## 1. 系统概述

本系统面向学校教务管理场景，涵盖教师、学生、课程、分数等核心实体，实现教学管理的数字化与流程化。

## 2. 核心实体与关系

### 2.1 实体关系图（ER 概要）

```
┌──────────┐       ┌──────────────┐       ┌──────────┐
│   教师    │──1:N─→│   课程        │←─N:1──│   学生    │
│ Teacher  │       │   Course     │       │ Student  │
└──────────┘       └──────┬───────┘       └─────┬────┘
                          │                      │
                          │      ┌──────────┐    │
                          └──N:M─→  选课记录  ←───┘
                                  │ Enrollment│
                                  └─────┬────┘
                                        │
                                  ┌─────▼────┐
                                  │   成绩    │
                                  │  Score   │
                                  └──────────┘
```

### 2.2 实体定义

| 实体 | 说明 | 核心字段 |
|------|------|----------|
| 教师 Teacher | 授课人员 | 工号、姓名、性别、职称、所属院系、联系方式 |
| 学生 Student | 在校学习者 | 学号、姓名、性别、年级、专业、班级、入学日期 |
| 课程 Course | 教学课程 | 课程编号、课程名称、学分、课时、课程类型、开课学期 |
| 选课记录 Enrollment | 学生选课关系 | 学生ID、课程ID、选课时间、选课状态 |
| 成绩 Score | 考核分数 | 学生ID、课程ID、平时成绩、期中成绩、期末成绩、总评成绩、绩点 |
| 院系 Department | 组织单位 | 院系编号、院系名称、院长、联系方式 |
| 班级 Class | 学生班级 | 班级编号、班级名称、所属专业、班主任、年级 |
| 专业 Major | 学科专业 | 专业编号、专业名称、所属院系、学制年限 |

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Id | long | 是 | 主键 |

| Semester | string(10) | 是 | 学期 |
| UsualScore | decimal | 否 | 平时成绩（0-100） |
| MidtermScore | decimal | 否 | 期中成绩（0-100） |
| FinalScore | decimal | 否 | 期末成绩（0-100） |
| TotalScore | decimal | 否 | 总评成绩（加权计算） |

> 总评成绩计算公式：`TotalScore = UsualScore × 0.3 + MidtermScore × 0.2 + FinalScore × 0.5`（比例可按课程配置）

## 4. 业务规则

### 4.1 选课规则

- 学生每学期选课学分上限为 25 学分，下限为 12 学分
- 必修课由系统自动预选，学生不可退选
- 选修课先到先得，达到最大选课人数后不可再选
- 退选窗口期为开课后 2 周内，逾期不可退选
- 有先修课程要求的，需先通过先修课程方可选课

### 4.2 成绩规则

- 成绩录入由授课教师负责，录入后不可自行修改（需申请修改）
- 总评成绩 60 分及以上为及格
- 绩点换算规则：

| 分数段 | 等级 | 绩点（4.0制） |
|--------|------|--------------|
| 90-100 | 优秀 | 4.0 |
| 80-89 | 良好 | 3.0 |
| 70-79 | 中等 | 2.0 |
| 60-69 | 及格 | 1.0 |
| 0-59 | 不及格 | 0 |

- GPA 计算：`GPA = Σ(课程绩点 × 课程学分) / Σ课程学分`
- 不及格课程可选择重修，重修成绩标注"重修"标记

### 4.3 毕业审核

- 修满专业培养方案要求的总学分方可毕业
- 必修课全部通过（含重修通过）
- 公选课学分满足最低要求
- 实践环节学分满足要求

## 6. 核心流程

### 6.1 选课流程

```
学生登录 → 浏览可选课程 → 检查先修条件 → 提交选课
    ↓
系统校验（学分上限/重复选课/人数限制）
    ↓
校验通过 → 选课成功 → 通知教师
校验失败 → 返回错误原因
```

### 6.2 成绩录入流程

```
教师登录 → 选择课程 → 录入各部分成绩 → 系统自动计算总评/绩点
    ↓
提交成绩 → 状态变为"已提交"
    ↓
学生可查看成绩
    ↓
如需修改 → 教师提交修改申请 → 管理员审批 → 审批通过后修改
```

### 6.3 毕业审核流程

```
管理员发起审核 → 系统自动检查每位学生的：
    - 总学分是否达标
    - 必修课是否全部通过
    - 公选课学分是否达标
    - 实践学分是否达标
    ↓
生成审核结果（通过/不通过/缺修清单）
    ↓
审核结果公示 → 学生确认 → 毕业证发放
```

## 7. 数据统计与报表

| 报表名称 | 说明 | 维度 |
|----------|------|------|
| 学生成绩单 | 单个学生所有学期成绩 | 按学期、按课程类型 |
| 课程成绩分布 | 单门课程成绩统计 | 分数段人数、平均分、及格率 |
| 班级成绩排名 | 班级内学生 GPA 排名 | 按学期、按学年 |
| 院系成绩对比 | 各院系平均 GPA 对比 | 按学期、按学年 |
| 选课统计 | 各课程选课人数统计 | 按学期、按课程类型 |
| 毕业率统计 | 各专业毕业率趋势 | 按年级、按专业 |
| 学业预警名单 | 不及格课程数或学分不足的学生 | 按学期、按院系 |

## 8. 非功能性要求

- **并发**：选课高峰期支持 5000+ 并发选课请求
- **响应**：页面加载 < 2s，成绩查询 < 1s
- **安全**：角色权限控制（RBAC），成绩数据加密存储，操作日志审计
- **备份**：数据库每日自动备份，成绩数据变更实时归档
- **可用性**：系统可用率 ≥ 99.5%
